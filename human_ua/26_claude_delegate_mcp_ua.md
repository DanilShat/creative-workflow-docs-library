# Claude як додатковий делегований оркестратор

## Ідея
Claude не стає ядром системи. Ядро лишається таким:

- сервер на ноуті оператора: стан, БД, LangGraph, локальна LLM;
- воркер на ноуті дизайнерки: браузери, Photoshop, After Effects;
- Claude Desktop на ноуті дизайнерки: опційний “розумний оператор” для складних творчих або Adobe-задач.

## Навіщо Claude
Локальна маленька модель добре підходить для дешевої рутини: класифікація задачі, вибір наступного кроку, створення job. Але Claude може бути кращим у задачах:

- розібрати, чому результат не влучає в ТЗ;
- запропонувати сильніший prompt repair;
- спланувати кілька кроків у Photoshop/After Effects;
- виступити як creative reviewer;
- допомогти дизайнерці в складних випадках, де локальний оркестратор занадто слабкий.

## Як Claude підключається
На ноуті дизайнерки піднімається локальний MCP server. Claude Desktop підключається до нього та бачить allowlisted tools:

- отримати контекст поточної задачі;
- подивитися доступні дії;
- запропонувати правку;
- створити запит на Photoshop/After Effects action;
- додати review note у workflow.

Claude не ходить напряму в базу і не керує сервером напряму. Він працює через наші безпечні інструменти.

## Що Claude може робити

- читати короткий контекст задачі;
- пропонувати план;
- викликати allowlisted tools через MCP;
- створювати review/fix notes;
- просити сервер створити job для Photoshop/AE дії;
- допомагати з prompt repair.

## Що Claude не може робити в MVP

- виконувати довільний JS/ExtendScript/UXP-код;
- напряму міняти БД;
- обходити worker protocol;
- тримати authoritative task state;
- замінювати Playwright browser flows;
- бути обовʼязковою залежністю для першого E2E.

## Правильний UX
У серверному UI може бути кнопка “Claude handoff”. Вона генерує пакет:

- що за задача;
- що вже зроблено;
- що не вийшло;
- які файли/артефакти є;
- які tools доступні Claude;
- що саме попросити у Claude.

Дизайнерка вставляє цей пакет у Claude Desktop. Далі Claude працює через MCP tools.

## Висновок
Claude варто додати як опційний потужний шар для складних сценаріїв, але перший робочий MVP має проходити без нього: Gemini + Freepik browser E2E через воркер.
